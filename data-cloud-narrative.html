SALESFORCE DATA CLOUD · UNIFIED INDIVIDUAL

配方時代 · Data Cloud 客戶統一資料層

客戶問：「Shopline / 蝦皮 / LINE / EDM / 廣告 資料真的在哪？如何做到跨通路 360 度？」
答：透過 Data Cloud 建立 Unified Individual DMO，即時聚合 5 大源系統，餵給 Customer 360 Console 與 Agentforce。

現況痛點

CURRENT STATE
配方時代目前每個通路的客戶資料都是各自為政： Shopline 有一組會員 ID、蝦皮買家帳號另一組、LINE User ID 又另一組、EDM 訂閱名單放在 Mailchimp、 FB / Google 廣告 CDP 又是另一份。行銷部要撈「近 30 天在蝦皮下單且開過我們 EDM 的客戶」時， 需要人工 join 三份 CSV，每次 2-3 天，時效已死。
DATA CLOUD 解
Data Cloud 用 Identity Resolution Ruleset（Email + Phone + LINE_User_ID） 自動把 5 個源系統的同一位客戶合成 1 個 Unified Individual， 並用 Calculated Insights 即時算出 LTV、活躍度、通路偏好， 透過 Data Action 反向寫入 Salesforce Person Account， 供 Customer 360 Console + Agentforce Prompt Template 讀取。

資料流動架構圖

5 源系統 → Data Streams → Identity Resolution → Unified Individual → Insights DMO → CRM / Agentforce

3 個關鍵 Calculated Insights (SQL / SAQL)

Data Cloud 特色：CRM 端不用寫 Apex，用 SAQL 直接在資料層跑 · 每 15 min 或 nightly refresh
💰 LTV_L365
SELECT
  i.Individual_Id__c,
  SUM(o.Order_Amount__c) AS LTV_L365__c
FROM Sales_Order__dmo o
JOIN Individual__dmo i
WHERE o.Order_Date__c >= today - 365
GROUP BY i.Individual_Id__c
近 12 個月的累積消費金額 · 客戶分層計算的主要依據
📉 Churn Score
CASE
  WHEN Days_Since_Last_Purchase > 180 THEN 90
  WHEN Days_Since_Last_Purchase > 90 THEN 60
  WHEN Interaction_Count_L30 < 2 THEN 40
  ELSE 10
END AS Churn_Score__c
流失預測分數 · 大於 60 → Customer_Tier__pc = 流失風險
📡 Preferred Channel
SELECT Individual_Id__c,
  MODE(Channel__c) AS Preferred_Channel__c
FROM Interaction_Event__dmo
WHERE Event_Datetime__c >= today - 90
GROUP BY Individual_Id__c
近 90 天最常互動的通路 · 用來決定送 LINE 還是 EDM
🎯 Reorder Signal
SELECT Individual_Id__c, Product_Id__c,
  (Purchased_Date + Suggested_Days) - today
  AS Days_Until_Depletion__c
FROM Sales_Order_Item__dmo x Product__dmo
依購買日 + 商品建議服用天數，反推斷貨日 · < 14 天觸發回購 alert
