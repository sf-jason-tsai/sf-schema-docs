Setup UI · 建 Agentforce Bot（10 分鐘）

1

Setup → Agentforce Studio → Agents → New Agent

  • Name：Thermos 客服 AI
  • Type：Service Agent（會自動掛 case、KM 存取權）
  • Description：處理 THERMOS 膳魔師客戶的零件詢問、保固維修、會員點數三大類客服案件
  • Company：皇冠金屬工業（THERMOS 台灣總代理）
  • Language：Chinese (Traditional)
Note：CBUSDO 若尚未看到「Agentforce Studio」，Setup 搜「Agent Builder」或「Einstein Copilot」也可以進去。SDO 通常已預先啟用。
2

建 Topic 1 · 零件詢問

  • Agent 頁面 → Topics tab → New Topic
  • Name：零件詢問
  • Classification：客戶詢問特定商品的相容零件、料號、單買方式、報價
  • Scope：僅處理零件相關詢問，不涉及保固、退換或會員相關問題
  • Instructions：「先用 SearchParts 找零件，若客戶同意加購再用 QuoteParts 建報價」
  • → Actions section → Add Action → 選：
    • THERMOS · 搜尋相容零件
    • THERMOS · 建立零件報價
3

建 Topic 2 · 保固維修

  • Name：保固維修
  • Classification：客戶反映商品故障、要求退換、詢問保固期、要求維修
  • Scope：處理 1 年保固判定、RMA 開立、物流排程。不處理零件加購或會員點數
  • Instructions：「先用 CheckWarranty 判定保固狀態，若保固內用 OpenRma 開單」
  • → Add Actions：
    • THERMOS · 判定保固狀態
    • THERMOS · 開立 RMA
4

建 Topic 3 · 會員點數

  • Name：會員點數
  • Classification：會員點數查詢、兌換碼異常、VIP 服務要求、要求補償點數
  • Scope：處理會員檔案查詢與點數調整。不處理零件或維修
  • Instructions：「加點數前先用 GetMember 確認會員身份，白金以上會員補償較高（150-200 點），一般會員 100 點」
  • → Add Actions：
    • THERMOS · 查詢會員檔案
    • THERMOS · 加值道歉點數
5

Activate Bot + 加到 Utility Bar

  • Agent 頁面右上 → Activate → 選擇 Thermos_Demo permission set
  • Setup → App Manager → Thermos app → Edit → Utility Items
  • Add Utility Item → Einstein Copilot / Agentforce → 選剛建的 Bot
  • Label：Agentforce Live、Icon：utility:einstein
  • Save → 到 Thermos app → 底部 utility bar 有 Agentforce Live 按鈕 → 點開就是官方 chat panel

Demo 話術（給皇冠 · Steven / 董座聽）

情境 1 · 對照展示（CustomLWC vs Agentforce）

「Steven，我先展示我 客製化 為皇冠做的 AI 面板——[點首頁 Business Chat] 這是我用 Salesforce Einstein Trust Layer 為你們寫的。3 週前開發，跟 Thermos 業務邏輯緊密整合。 現在我打開官方 Agentforce Bot——[點 utility bar 「Agentforce Live」] 同一組後端 Action（零件搜尋、保固判定、加點數），Salesforce 官方框架就內建這種對話能力。您導入之後，客服不用等我們客製，可以在 Agent Builder 用滑鼠拖拉——加 Topic、換 prompt、串新 Action，全部 IT 自己搞定。」

情境 2 · 實測對話（讓客戶親眼看到 Agent 決策）

輸入：「我有一個 JNL-504 的保溫瓶，墊圈想單買」 Agent 內部推理（Setup 開 debug 可看）： 1. Planner → 判斷屬於「零件詢問」Topic 2. 選 Action：THERMOS · 搜尋相容零件 3. 從訊息抽出 modelCode = "JNL-504" 4. 呼叫 Apex → 回傳 4 個零件 + 價格 5. 生成自然語言回覆 + 若客戶要加購再叫 QuoteParts Agent 回覆客戶：「JNL-504 系列有 4 款相容配件：JNL-504 專用墊圈（NT$ 80）、One-Touch 杯蓋（NT$ 220）、清潔刷組（NT$ 150）、專用吸管（NT$ 60）。全部加購 NT$ 510。要幫您開報價單嗎？」

情境 3 · 商業價值收斂

「Steven，這個 Bot 目前 3 個 Topic × 2 Action——**上線後你們 IT 可以無限延伸**。 想加「電商訂單物流查詢」？寫一個 @InvocableMethod 查 Order + THL API，掛到新 Topic。不用重寫 chat UI，Salesforce 官方會幫你 handle 對話流。 計費：$0.10 per conversation。你們現在客服 5K case/月，換算 NT$ 15,000/月——AI 接手 tier-1（零件詢問、會員點數這種標準流程）省下 6 個客服的 30% 工時，一個月省 NT$ 72,000 人力成本。5 倍 ROI。」
