建商-EE · App Overview

建商-EE App · 對客戶說明

建商-EE 是 CBUSDO 上第二套「建商」demo，跟既有的「建商-ProSuite」平行共存。 透過 建商-EE · 前綴、專屬 List View / Report / Dashboard 完全隔離。

App 結構

建商-EE (EE_Real_Estate) ├── 🏠 Home ← 客製 Home Lightning Page (EE_RE_Home) ├── 🏢 建案-EE (Building_Project__c) ← Custom Object · 9 個欄位 ├── 👤 Lead ← Standard，過濾 建商-EE · % ├── 💼 Opportunity ← Standard，過濾 建商-EE · % ├── 🏛 Account ← Standard，過濾 建商-EE · % ├── 👥 Contact ← Standard，依 Account 過濾 ├── 📋 Case ← Standard，過濾 建商-EE · % ├── 📊 Reports ← EE_RE_Reports folder └── 📈 Dashboards ← EE_RE_Dashboards folder

Home Page (Custom Lightning Page)

兩個 App 各自綁定一份專屬 Home：

  • EE_RE_Home（label「建商-EE Home」）— 綁 建商-EE
  • ProSuite_RE_Home（label「建商-ProSuite Home」）— 綁 建商-ProSuite（本次順便補回）

兩個 Home Page 都有：Rich Text 歡迎語 + 嵌入專屬 Dashboard + Recent Records + Today's Task + Event Container。

⚠️ 手動步驟：Salesforce Metadata API 不支援用 script 綁 Home Page 到 App。請進 Setup → Lightning App Builder → 開 EE_RE_Home（或 ProSuite_RE_Home）→ Activation → Assign as App Default → 選對應 App → 存檔。約 30 秒。

Permission Set · EE_Real_Estate_User

  • App visibility: EE_Real_Estate = Visible
  • Building_Project__c: 完整 CRUD + View All / Modify All
  • Tab: Building_Project__c = Visible
  • Field Permissions：全部 9 個 custom field + 6 個 EE_External_Id__c 明確列出 readable/editable = true（避免 SOQL 「No such column」的 FLS 陷阱）
