3 · 遇到的坑 (Postmortem)

坑 #1 · Duplicate Rule 擋 Lead insert

第一次 upsert Lead 有 6 筆被擋（DUPLICATES_DETECTED），第二次加 +ee email 後還剩 3 筆（可能是 Phone 撞到）。

解法：對這 3 筆用 REST API /services/data/v67.0/sobjects/Lead 加 header Sforce-Duplicate-Rule-Header: allowSave=true 逐筆 insert。

sf api request rest /services/data/v67.0/sobjects/Lead \
  --method POST \
  --header "Sforce-Duplicate-Rule-Header: allowSave=true" \
  --body @lead.json --target-org CBUSDO

坑 #2 · IsConverted Lead 無法直接 insert

Salesforce 平台限制：insert 時不能設 IsConverted=true。EE org 上有 6 筆已轉換的 Lead（王底迪、林單、許琳琳、紀小華、林子後、王小迪）。

解法：全部 17 筆先以「未轉換」狀態 insert；demo 時如需展示這 6 筆的 convert 動作，手動再演一次。

坑 #3 · CustomReportType & Report grouping 失敗

原計畫要建 3 個 report（Pipeline / Sales_Rank / Property_Progress）+ 4-tile Dashboard。實際部署時：

  • USERS.NAME 和 OWNER grouping 都被拒（invalid value）
  • Building_Project__c 沒有預設 report type，自訂 CustomReportType 又踩 NAME field syntax 錯誤

解法：簡化到只保留 EE_Pipeline 一份 report，Dashboard 3 個 widget 全都用它當資料來源。日後有需要可再補其他 report。

坑 #4 · Bulk API LineEnding = CRLF

Node 產出的 CSV 用 LF，sf CLI bulk upsert 要求 CRLF。首次 Account upsert 直接 fail。

解法：sed -i 's/$/\r/' data-import/*.csv 統一轉 CRLF。

坑 #5 · sf query 走 pipe 會夾雜「Querying Data... done」

sf data query --result-format csv > out.csv 會把 stderr 混進去。

解法：一律加 2>/dev/null。
