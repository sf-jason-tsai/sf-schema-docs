不在 Individual 上加 LINE_UID／Oracle_Member_ID 客製欄位 · 用 Party Identification 存不同 ID type · 這是 Salesforce 官方最佳實踐 · 未來要加入 momo Buyer ID、蝦皮 UID、抖音 OpenID 都只是新增 PartyIdentification type · Individual schema 永遠乾淨。

4RFM Calculated Insight SQL

Data Cloud Calculated Insight · 每小時執行一次 · 產出 Insight_RFM__dlm DMO，可直接在 Segmentation Builder 拉條件。

SELECT
    UnifiedIndividualId__c AS IndividualId,
    DATEDIFF('day', MAX(so.OrderDateTime__c), CURRENT_DATE) AS Recency__c,
    COUNT(DISTINCT so.OrderNumber__c) AS Frequency__c,
    SUM(so.TotalAmount__c) AS Monetary__c,
    AVG(so.TotalAmount__c) AS AvgOrderValue__c,
    COUNT(DISTINCT sop.IpLicense__c) AS IpBrandDiversity__c,
    CASE
        WHEN SUM(so.TotalAmount__c) > 50000 AND COUNT(*) >= 6 THEN 'VIP'
        WHEN DATEDIFF('day', MAX(so.OrderDateTime__c), CURRENT_DATE) > 180 THEN 'Sleeping'
        WHEN DATEDIFF('day', MAX(so.OrderDateTime__c), CURRENT_DATE) > 90 THEN 'AtRisk'
        ELSE 'Active'
    END AS RFM_Segment__c
FROM ssot__SalesOrder__dlm so
JOIN UnifiedssotIndividual__dlm u ON so.SoldToCustomerId__c = u.SourceRecordId
JOIN ssot__SalesOrderProduct__dlm sop ON sop.SalesOrderId__c = so.Id
LEFT JOIN ssot__Product__dlm p ON sop.ProductId__c = p.Id
GROUP BY UnifiedIndividualId__c

使用方式

  • 每小時執行一次 · 產出可查詢的 Insight_RFM__dlm
  • Segmentation Builder 可以直接把 Recency__c／Frequency__c／Monetary__c／IpBrandDiversity__c／RFM_Segment__c 當篩選條件
  • 作為下方 6 個 seeded segments 的核心依據
