Demo Beat Sequence

1

現況痛點

開場 · 建立 pain 共識
2 min
  • 開場破題：「有你共創目前有 Oracle 會員 DB、LINE OA、官網三套會員系統各自為政，行銷部要撈跨渠道名單得靠 IT 人工 join CSV」
  • Screen：App Launcher 打開「有你共創 · Marketing Cloud」App
  • Highlight：首頁 4 個 KPI Tile (Total Members / Active Segments / Revenue Lift / Journey Live) 加上 6 個 segment 縮圖卡
  • 引導提問：「行銷部現在要撈跨渠道 VIP 沉睡客名單，要多久？」
  • 預期回答：「3 天 · IT 幫忙從 3 個系統匯出 CSV、人工比對 email、去重複」
今天要證明的一件事 — 讓 marketer 自己動手，把這 3 天縮到 3 分鐘，而且不用另外買 Marketing Cloud tenant，全部在你們既有的 Salesforce Core org 裡完成。
2

Unified Profile 360 · Data Cloud aha

Identity Resolution 破碎顧客拼一起
4 min
  • Navigate：點 賴雅筑 record 進入 Customer 360 頁
  • Layout 左區：3 筆 source rows 並列顯示
    • Oracle POS row：lai_yj@yahoo.com · 0912345678 · 賴雅筑
    • LINE OA row：LINE UID U3f8a... · 電話 +886-912-345-678 · Nickname 雅筑
    • 官網 Contact row：laiyj@ycc.com.tw · 桃園市 · Christy Lai
  • Layout 中區：Identity Resolution 4 條 match rules 動畫 (Email fuzzy / Phone normalize / Name similarity / Behavior signal)
  • Layout 右區：合成 Unified Individual 卡 · 標籤 「RFM: Sleeping · LTV NT$62,000 · 最愛 IP: Disney-Duffy · 上次消費: 210 天前」
這就是 Data Cloud 的 identity resolution — 三個系統、三個不同的 email 格式、三個電話格式，Data Cloud 用 match rule 拼成一個真人。以前這是 IT 一個月專案，現在是 out-of-the-box。
3

Segment Builder

Marketer 自助 · 3 分鐘撈名單
4 min
  • Navigate：切到「智慧分眾」tab · 6 segment 列表 (高價值沉睡客 / 新品搶先族 / 快閃店常客 / 跨 IP 母嬰客 / momo 高頻客 / Z 世代盲盒族)
  • 點擊：「高價值沉睡客」segment card
  • 顯示：Data Cloud Calculated Insight SQL preview (LTV > 50K AND Last_Order > 180 days AND Member_Level IN ('金卡','VIP'))
  • Members preview：表格顯示 8,542 名會員 · 賴雅筑排第 3 · 命中！
  • Action：按「發佈到 Activation」按鈕 → dialog 選 Meta + Email + SMS → 確認送出
重點：這整個過程 marketer 不用寫 SOQL、不用開 case 給 IT、不用等 3 天。segment definition 是 metadata，可以 clone、可以版本控管、可以 CI/CD deploy 到 production。
4

Campaign Flow · MCoC on Core

Salesforce Flow 就是 Journey Builder
4 min
  • Navigate：切到「旅程編排」tab · 6 個 journey canvas 縮圖
  • 展開：「沉睡喚回」flow · 全螢幕 canvas 檢視
  • Flow 節點順序：
Segment Trigger→ Wait 1 Day→ Send Email · 「我們想您了」→ Wait Until Open (72h)→ Decision · Opened?
  ├─ Yes → Send SMS · Duffy 新品 8 折碼→ Update Member.Status = Reactivated
  └─ No  → Send Email retry · 個人化 Duffy 推薦→ Update Member.Status = Follow-up
這是 Salesforce Flow — 是你們 admin 已經會用的工具。Marketing Cloud Growth (on Core) 用 Flow 作為 journey engine，不是傳統 SFMC Journey Builder，不用學 AMPscript、不用另外開 tenant。Marketer 培訓 2 週就上手。
5

廣告 Activation

跨 5 個目的地 · 一鍵送出
3 min
  • Navigate：切到「廣告激活」tab
  • 5 個 activation target cards：
    • Meta Advantage+ · LIVE · Audience 8,542 · 上次同步 2 分鐘前
    • Google Customer Match · LIVE · Audience 6,120
    • LINE Ads Platform · ARCHITECTURE ONLY · 需 MC Engagement + LMC + LAP 三跳
    • Email (Marketing Cloud Growth) · LIVE · 上次發送 8,540 封
    • SMS (Every8d 台灣簡訊) · LIVE
  • 展開：SVG 架構圖 · Segment → Data Cloud Activation → 5 個 targets 的資料流
  • 誠實揭露 LINE 限制：「LINE Ads Platform 目前不是 native connector · 需要透過 MC Engagement 中繼、經由 LINE Marketing Console (LMC)、再到 LINE Ads Platform (LAP) 三跳 · 這是台灣市場現況、不是我們一家的限制」
誠實一點很重要 — 我們不會為了 demo 假裝 LINE 是 native。但這個限制未來 12 個月會被 Segment CDP connector 補上，你們現有的 org 架構不用改。
6

ROI 回讀 + IP 品牌分析

收尾 · 從數據看策略決策
3 min
  • Navigate：切到「IP 品牌分析」tab
  • 10 個圖表快掃：
    • ① IP 品牌營收長條圖 · ② 城市熱力地圖 · ③ 渠道分布甜甜圈
    • ④ 月度趨勢線圖 · ⑤ 會員等級 × IP 交叉表 · ⑥ 品類交叉購買 sankey
    • ⑦ 快閃店 vs 官網 AOV 對比 · ⑧ momo 高頻客 cohort retention
    • ⑨ RFM 4 象限散點圖 · ⑩ 新品首週轉化漏斗
  • 回首頁：KPI Tile 動起來 · 顯示「這 6 個 segment 帶動的 Revenue Lift 合計 NT$4,000,000」
這個數字不是模型預測 — 是 Data Cloud 直接回讀 Sales Order 對照 Segment membership 算出來的實際 lift。CMO 隔天 board meeting 就能拿這一頁去講故事。

Ending · 3 個 Closing 訊息

  • 未來 Agentforce Marketing 直接在同一個 org 內長出 · 一次投資、持續拿到 Salesforce 每季的新能力 · 不用重新 POC
  • 不用另外開 Marketing Cloud tenant · 不用學 AMPscript · Marketer 只要會用 Flow · 培訓成本降 80%
