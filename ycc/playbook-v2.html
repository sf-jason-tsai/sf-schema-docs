3. MarketSegment Activation 建立步驟（Data 360 UI）

目標：讓 9 個 Segment 能真的送到 Journey / EDM，不只是靜態名單。

1
登入 FirstSDO → App Launcher → Data Cloud（對客戶介紹時說「Data 360」）
2
左側導覽 → Segments → 選 有你共創 · 白金 VIP
3
頁面右上角 Publish Now → 等 30 秒完成 → 頁籤切到 Activations
4
New Activation → 選 Activation Target：Marketing Cloud on Core（Email）
5
Attribute mapping：勾選 FirstName / LastName / Email / MemberTier__c / PointsBalance__c → Save
6
對 沉睡會員、雙11生日壽星、千禧世代女性文創迷 重複 —— 這 4 個是 demo 必用
順序重要：Segment 必須先 Publish 一次，Activation UI 才會出現該 Segment 的下拉選項。

6. AI-driven 差異化話術（Einstein / Agentforce）

MC Advanced 的殺手鐧 = AI 加值。這場 demo 對決策者 Edgar，AI 是「你們比別家強在哪」的關鍵一段。

可以帶到的三個 AI 話術點（不用做，只要講）

能力怎麼介紹Edgar 會聽到什麼
Einstein Segment Recommendations 「我們平台會自動根據您的 60 萬會員行為，推薦『看起來像白金 VIP 但還沒消費過』的高潛客名單。」 不用資料科學家也能做 lookalike
Einstein Send Time Optimization 「這封雙 11 EDM，每個人收到的時間是 AI 算出來的最佳開信時段——同樣的內容，開信率會多 15-25%。」 直接影響轉換率的數字
Agentforce Marketing Copilot 「行銷同仁直接對 Agentforce 說：『幫我找上個月加購但沒結帳的女性會員，發封文創桌遊的 EDM』——它會自己建 Segment、生成 EDM 主旨、綁 Journey。」 把 3 天工作變 5 分鐘的路線圖預覽
誠實原則：這 3 塊都是產品路線圖上的能力，不是這場 demo 現場能點出來給 Edgar 看的畫面。話術中用「路線圖預覽」帶過，不要現場 demo 導向 Einstein 頁面（會撲空）。

7. 80 萬會員規模的視覺呈現

Demo 現場只有 60 筆真實資料，客戶目標是 80-100 萬會員。UI 上要處理：drill-down 可看到真實名單，同時大數字用示意規模，兩者用 SLDS info icon 分開說明（符合誠實原則）。

uniicreativePortal LWC 的頂部大數字建議設計

┌─────────────────────────────────────────────────────────────┐
│ 會員總數 802,431 ⓘ  ← info icon: "demo 資料放大顯示，       │
│                        底層抽樣 60 筆真實 Contact"          │
│                                                              │
│ 白金 VIP  金卡    銀卡    一般會員                          │
│  80,412   80,412  214,432 427,175                            │
│                                                              │
│ 上週活躍 154,802     沉睡會員 (180d) 121,065                │
└─────────────────────────────────────────────────────────────┘

比例保持 60 筆真實資料的分布（白金 10% / 金 10% / 銀 27% / 一般 53%），乘一個放大係數就好。這樣客戶點進 drill-down 名單，仍會看到跟大數字比例一致的真實會員。
