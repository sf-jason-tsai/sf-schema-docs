MIGRATION REALITY
傳統 MC → MCoC 沒有 lift-and-shift 工具
Salesforce 明確支援 雙軌並行 (dual-run) 策略：既有客戶的傳統 Marketing Cloud Engagement 可繼續使用，並在同一組織內同時導入 MCoC 服務新的業務線或品牌。
官方立場：Rebuild, not migrate. 沒有自動轉換 Journey / AMPscript / Data Extension 的工具。所有 asset 需在 MCoC 上以 Flow + Data Cloud DMO 邏輯重新建構。
對 有你共創 而言，這 反而是優勢：作為新專案，直接 native 在 Core 上啟動，無 legacy 包袱。
PART 2 · STRATEGIC RATIONALE

為何 有你共創 選 MCoC

四個決策關鍵，來自 有你共創 多品牌 IP 授權經銷 (Sanrio · Labubu · Disney · momo) 的實際營運需求。

1

一個 org 打通行銷 · 客服 · 銷售

Marketer 看到的顧客 = 客服看到的顧客 = 銷售看到的顧客。沒有 context switch、沒有跨 tenant 資料同步延遲。客服接到 Labubu 快閃店退貨電話時，直接看到該顧客上週剛收到什麼 email、點過什麼 SMS。

2

Data Cloud + Person Account + Flow 全 native

Marketer 直接在 Core 上 build Segment，用熟悉的 Flow builder 設計 Journey，不需要學 AMPscript 或 SSJS。Person Account 天然支援 B2C 商業模式，多品牌會員在同一顧客身上 aggregate。

3

Trial Org 直接 provisioning · 零額外採購

CBUSDO Trial Org 已 provision Marketing Cloud on Core，不需申請另一個獨立的 MC tenant、不需另一組 admin 帳號、不需要走跨系統 SSO 整合流程。從 kickoff 到第一封 email 發出時間縮短 60%+。

4

Agentforce Marketing 天生在 MCoC 上長

Salesforce 未來 AI 行銷 agent 的所有能力 (自動生成 segment / 自動優化 subject line / 自動 A/B) 都是先在 Core 版落地。選 MCoC = 直接站在 Agentforce 主線上，未來擴充 AI Agent 不需要重新架構。

PART 4 · TECHNICAL REFERENCE

Campaign Flow Elements · 可用 / 不可用

MCoC Campaign Flow (Segment-triggered) 支援的 Flow element 清單。與傳統 Screen Flow / Record-triggered Flow 不完全相同。

✓ 可用 Elements

  • Decision · 分支判斷邏輯
  • Loop · 集合迭代
  • Wait · 固定延遲 (小時 / 天)
  • Wait Until Event · 等待 open / click / purchase
  • Send Email · MCoC 原生 email action
  • Send SMS · SMS action (透過整合 provider)
  • Create Record · 建立 SF record
  • Get Record · 查詢 record
  • Update Record · 更新 record (含 DMO write-back)
  • Delete Record · 刪除 record
  • Collection Filter · 集合過濾
  • Collection Sort · 集合排序

✗ 不可用 Elements (Segment-triggered)

  • Screen · 沒有 UI · Segment flow 是背景執行
  • Subflows · 尚未支援 nested flow
  • Apex Actions · Custom Apex invocable 未開放
  • Custom Errors · 錯誤訊息 UI 元件
  • Pause / Wait Configurations · 進階 pause 條件
  • Platform Events · 直接 publish event 尚未開放
PART 5 · TERMINOLOGY NOTE

為何全文使用 "Marketing Cloud Growth (on Core)"

TERMINOLOGY LOCK
Salesforce 官方最不引起歧義的稱呼
Salesforce 生態內部有多個代號指涉同一產品線：
❌ "MC Next" — 早期內部代號，客戶溝通易誤解為未來版本
❌ "Agentforce Marketing" — AI Agent 品牌名，範圍比 MCoC 更廣、包含跨產品 agent
❌ "MC Growth" (單獨) — 只指 Growth Edition · 不含 Advanced Edition
✓ "Marketing Cloud Growth (on Core)" — 官方在客戶溝通、Trailhead、Ideas Exchange 上最常用的表述，明確定位「在 Core 上、行銷雲產品線」

本 demo 全文統一鎖定此稱呼，包含 slides、PRD、Flow metadata description、Rollout Playbook — 避免利害關係人在會議中因用詞混淆產生方向偏移。
